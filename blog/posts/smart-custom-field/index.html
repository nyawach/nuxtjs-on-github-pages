Smart Custom Fieldを使ってカスタムフィールドを追加する

INDEX

    WordPressの案件で必ずつきまとうのが、カスタムフィールド管理について。 今回は、カスタムフィールドの設定をコードとして保存できるSmart Custom Field(SCF)についてと、カスタムフィールドを追加する手順についての記事です。

    Smart Custom Fieldについて

    カスタムフィールドをコードベースで設定できる WordPressプラグインです。 DBに保存する必要があるカスタムフィールドの設定項目ですが、このプラグインを使うことで .php ファイルで管理することができます。 基本的なループやラジオボタンなどは対応しています。(関連付けなどはなかったかも) 開発者が日本の方だそう。

    Smart Custom Fieldを使うメリット

    Git管理ができる

    これに尽きると思います。 また、バージョン管理せずに他のプラグインのように管理画面から設定することも可能です。 ただし、そちらで設定した項目はDBに保存されるため、バージョン管理の項目から外れてしまいます。

    ループに対応

    グループごとにループの設定ができます。 また、デフォルトではないループで設定した値をグルーピングして取得するための機能(SCF::get('group-name'))が用意されています。

    今回は、ラーメンを投稿するブログサイトを作る場合に、以下のような項目を入力してもらう想定でカスタムフィールドを指定しましょう。

    Smart Custom Fieldをインストール&有効化

    管理画面などからプラグインを検索してインストールしましょう。

    ターミナルからなら、WP-CLIでインストールできると楽ですね。

    wp plugin install custom-post-type-permalinks --activate

    実際にコードを書く

    今回は、post 投稿タイプにのみ表示されるカスタムフィールドを作成します。

    function.phpにカスタムフィールドの項目を追加する関数を書く

    function add_shop_info_fields($settings, $post_type, $post_id, $meta_type) {
  // post以外の投稿タイプなら特に何もしない
  if($post_type != 'post') return $settings;

  // 引数: (設定名スラッグ, ラベル名)
  $Setting = SCF::add_setting('shop-info', '店舗情報');

  $items = array(
    array(
      'name'  => 'name',
      'label' => '店舗名',
      'type'  => 'text'
    ),
    array(
      'name' => 'place',
      'label' => '住所',
      'type' => 'textarea',
    ),
    array(
      'name' => 'description',
      'label' => '店舗の説明',
      'type' => 'textarea',
    ),
    array(
      'name' => 'image',
      'label' => '店舗の画像',
      'type' => 'image',
    ),
  );

  // 引数: (グループ名, ループの可否, 項目の配列)
  $Setting->add_group('group-shop-info', true, $items);

  // 今あるカスタムフィールドの設定に$Settingを追加
  array_push($settings, $Setting);

  return $settings;
}

// smart-cf-register-fieldsにフック
add_filter('smart-cf-register-fields', 'add_shop_info_fields', 10, 4);

    これを保存して新規投稿画面に移動すると、下の方にカスタムフィールドが表示されていると思います。

    ![画像]

    該当ページに設定した情報を表示させる

    基本的には、メインループ内で SCF::get('meta_data') で取得することができます。

    <?php
  $info_array = SCF::get('group-shop-info');
  foreach($info_array as $info):
?>
  <?php if(isset($info['name']): ?>
    <p><?=$info['name'] ?></p>
  <?php endif; ?>
<?php endforeach; ?>

    該当ページ以外でカスタムフィールドの値を取得する場合

    たとえば、トップページに店舗名の一覧だけ表示したいという場合には、カスタムフィールドを設定したページのID(固定ページなら固定ページのID、投稿なら記事IDなど)を設定することで取得することができます。

    <?php
// front-page.phpなど
SCF::get('group-shop-info', get_page_by_path('shop'));

    SCFを使う時に覚えておくこと

    ループの有無で取得方法が変わる

    画像/ファイル/ページなどはすべてIDが保存される

    WordPressにアップロードした画像などのファイルや、関連記事などで選択した記事をSCFで設定した場合、すべてIDとして保存される仕組みになっています。

    言及してたURL

    したがって、ファイルや記事を取得したい場合は、 wp_get_attachment() や get_the_post() などにIDを渡して取得する必要があります。 統一されているのでわかりやすいです。

    カスタムフィールドの投稿タイプの種類

    基本的にWordPressのデフォルトで指定できるものには対応しています。
